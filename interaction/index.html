    Interaction Model

    Polydesk uses a modal, lazygit-style navigation layout: the left column is the primary interaction surface, and the right side renders context for the focused navigation pane.

    ┌────────────────────┬────────────────────────────────────┐
│  NAVIGATION        │  CONTEXT VIEW                      │
│  1 Watchlist       │  Market Detail                     │
│  market rows       │                                    │
│  2 Candidates      │  Order Book                        │
│  market rows       │                                    │
│  3 Orders          │  or pane-specific content          │
│  4 Account         │                                    │
│  5 Strategies      │                                    │
└────────────────────┴────────────────────────────────────┘

    Theme

    All colours use the Catppuccin Frappé palette via curses.init_color(). The 8 standard curses colour indices are reassigned to precise Frappé RGB values at startup, giving a consistent look across terminals. If a terminal does not support can_change_color(), the TUI falls back to the standard curses palette.

    Normal Mode

    Normal mode is for navigation-pane movement, row selection, and context scrolling.

    Key Action
    Left / Right Move focus to previous / next navigation pane when navigation owns focus
    1–6 Jump directly to Watchlist, Candidates, Orders, Account, Strategies, or Discovery
    Enter / Tab Enter or cycle the right-side Market Detail / Order Book scroll focus
    Shift+Tab Cycle the right-side scroll focus backward
    Esc Return from right-side scroll focus to navigation focus
    Up / Down In Watchlist/Candidates: select a market. In right-side focus: scroll content
    [ / ] When the full Chart workspace has focus, decrease / increase its common range presets
    g / G First / last market in the focused market pane
    x Remove the selected Watchlist market while Watchlist navigation has focus
    / Enter command mode with search prefilled
    : Enter command mode
    ? Toggle help

    Focus indicator: the left navigation border is bright while navigation owns focus. Selected market rows are highlighted only when their navigation pane owns focus. Enter or Tab can temporarily move focus to Market Detail or Order Book for scrolling; Esc returns to navigation focus.

    Command Mode

    Press : to enter command mode. The command picker uses fuzzy matching and shows a vertical suggestion popup. Tab cycles suggestions, Up/Down moves through suggestions and scrolls the list when needed, Enter runs the selected command, and Esc cancels command mode. A command with no match remains visible with an explicit error prompt until it is submitted or cancelled.

    Command names support aliases and unique-prefix abbreviations. For example, :s, :search, and a unique fuzzy match for search resolve to the same command.

    Core Commands

    Command Aliases Action
    :search [filters] <query> :s, :find Search markets using composable lifecycle and availability filters
    :limit [1-100] :lim, :results-limit Show or set the candidate result limit; default is 30
    :recommend :rec, :recommended Load recommended open-market candidates
    :watchlist :w, :watch Focus Watchlist
    :candidates :c, :results Focus Candidates
    :pane <name> :nav, :p Focus watchlist, candidates, orders, account, or strategies
    :workspace <name> :ws Compatibility alias for pane focus
    :add <n|id|slug> :a Add a candidate to Watchlist
    :remove <n|id|slug> :rm, :delete Remove a watched market
    :account [<0x-address> [label]\|clear] :acct Show Account, save a public account identity, or clear it
    :refresh :ref, :reload Refresh the current market view
    :yes :y Show the YES order book
    :no :n Show the NO order book
    :chart [<number>s\|m\|h\|d\|w\|max\|all] :ch Replace the market context with a full Bid / Ask / Deal chart
    :book :ob, :orderbook Return to the Market Detail and Order Book depth layout
    :market :m, :open Show selected market identifiers
    :help :h, :? Toggle help
    :quit :q, :exit, :wq Exit Polydesk

    CLOB Commands

    These commands operate on the currently selected market and active YES/NO token.

    Command Aliases Action
    :mid :midpoint Show midpoint price
    :spread :spr Show bid-ask spread
    :history [interval] [fidelity] :hist, :ph Show price history; default is 1d 30
    :negrisk :nr Show negative-risk status
    :detail :dt Show full CLOB detail for the active token

    Discovery Commands

    Command Aliases Action
    :events [tag] [limit] :ev List active events, optionally filtered by tag
    :event <id> :e Load an event and show its markets in Candidates
    :series [limit] :ser Open the Discovery workspace with a bounded Series page
    :tags :tag List tags
    :disc :discovery Focus the Discovery workspace

    Command Aliases Action
    :leaderboard [period] [order_by] [limit] :lb, :top Show trader leaderboard output
    :profile <address> :pf, :user Look up a public profile
    :comments <event|market> <id> :cmts List comments for an event or market
    :holders [condition_id] :hld Show top token holders; defaults to selected market condition ID

    :evolume <event_id> :evol Show event volume

    Active List

    When Watchlist or Candidates is focused, the right-side context view shows Market Detail and Order Book for the selected market. Search/recommend focuses Candidates; adding/removing markets returns focus to Watchlist when appropriate. Closed watched markets are marked CLOSED in red in the navigation list. Text output and price history temporarily occupy the lower right-side slot normally used by Order Book. Orders and Strategies are routed read-only panes for now.

    Account Identity

    The Account pane displays the optional local public address and label saved with :account <0x-address> [label]. It is deliberately an identity/configuration view, not an authenticated wallet view: it does not load .env, request a signature, access balances, or store any secret. :account clear removes the saved identity.

    Discovery Workspace

    Series, Events, and Tags use the dedicated Discovery workspace (6) rather than rendering below Candidates. Its list owns Up/Down and Enter: selecting a Series opens a bounded Event page, selecting an Event loads its Markets into Candidates, and selecting a Tag opens its Events. Press Esc from a Series Event page to return to the preceding Series page and its selected row; press Esc again from the root Discovery page to return to the navigation pane that opened it. A search or recommendation immediately returns focus to Candidates without deleting the current Discovery page.

    Discovery retains only one current page of at most 25 lightweight items plus, only while drilling from a Series to Events, one 25-item return snapshot. Each root discovery request replaces that state; no unbounded navigation history, Series payload cache, or duplicate Market list is kept in memory.

    Market Search Filters

    :search <query> searches open, active markets. Filters go before the query and can be combined across groups:

    :search -t --unrestricted election
:search -c --archived "NBA Finals"
:search -a -r --no-book bitcoin
    Group Flags Meaning
    Lifecycle -o / --open, -c / --closed, -a / --all Open is the default; choose closed or every lifecycle state explicitly.
    Trading -t / --tradable, -x / --suspended Tradable requires active, open, accepting orders, and an enabled order book. Suspended means open but not accepting orders.
    Access -r / --restricted, --unrestricted Include only region/compliance-restricted markets, or only unrestricted markets.
    Order book --book, --no-book Require an enabled CLOB order book, or its absence.
    Catalogue --archived / -arch, --unarchived Filter by archive visibility.
    Deployment --ready, --pending Require a ready market, or a pending/deploying market.
    Resolution --resolved, --unresolved Require a resolved market, or a closed market still awaiting resolution.
    Exact ID -id <market_id>, --id <market_id> Load one market directly by exact API ID, including closed markets; do not add a keyword query.

    Flags in the same group are mutually exclusive; flags in different groups are intersections. -t and -x are only meaningful for open markets, so they cannot be combined with -c. :search -a <query> uses the same search source as ordinary :search <query> and only changes the lifecycle result filter to all states. Other explicit filters use one bounded highest-volume market-list page and apply the query and filter intersection locally.

    Transient Gamma failures during market-list or exact-market requests retry up to three times with short exponential backoff. Invalid requests are reported immediately without retries.

    Candidates default to 30 results. Use :limit 45 to change the current session's limit within 1–100; running it after a search immediately repeats that search at the new limit. The left Candidates list scrolls to keep the selected market visible.

    When the first command token is still ambiguous, Command mode keeps its full fuzzy suggestion list. Once a command plus arguments uniquely identifies a command, such as :s fal, the picker collapses to search and shows that command's exact usage line. Entering :search, :s, or :find opens a compact contextual filter panel under the usage line, with lifecycle, trading, access, order-book, archive, and deployment flags grouped by their mutual-exclusion category.

    Live Bid/Ask Chart

    The full Chart workspace combines available trade-price history with locally retained Best Bid / Best Ask samples for the active YES or NO token:

    • Green B line: highest available bid.
    • Red A line: lowest available ask.
    • Cyan History line: historical trade prices from CLOB. It is not a reconstructed historical bid or ask.
    • Yellow Deal line: the most recent trade price returned with each live order-book snapshot.
    • Live refresh interval: 2 seconds, using read-only polymarket clob book <token> -o json polling.

    Run :chart to open the full workspace. It uses thin Braille lines for the cyan trade-price baseline, green Bid, red Ask, and yellow Deal series. :chart 10m, :chart 2h, :chart 3d, and :chart 1w accept any positive duration with s, m, h, d, or w; :chart max and :chart all show all available history. Without an argument it uses 15m. With Chart focused, [ and ] move through common presets from 30s through max. Samples arrive every 2 seconds and remain available for the selected token until the market or YES/NO side changes. The history worker selects an endpoint range that covers the chart window, then the chart keeps only the requested range. Use :book to restore the depth table. The graph is terminal-rendered rather than a Matplotlib bitmap, but its time and price coordinates use the source values directly. When CLOB reports that a token has no order book, the panel shows an unavailable state and waits 60 seconds before retrying automatically; use :ref to retry immediately.

    Depth Ladder

    The standard Order Book view presents the best asks and bids side by side on each depth row. Ask size/price appear on the left in red; bid price/size appear on the right in green. This keeps the two sides comparable without scrolling between separate sections.

    Use :chart for a full-height chart workspace. It replaces both Market Detail and Order Book depth with the live bid/ask chart so that the time and price axes have the full right-side workspace. Use :book to restore the standard market detail and depth layout. The chart remains terminal-rendered, so it is numerically time/price accurate but does not emulate Matplotlib's pixel-level anti-aliasing.

    Bottom UI

    • Statusline: current mode, active list + position, pending work, refresh cadence.
    • Promptline: command input or short status/error messages.
    • Completion strip: command suggestions shown above the statusline while typing.
